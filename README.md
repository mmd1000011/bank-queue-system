#سیستم نوبت‌دهی بانک (Bank Queue System)
.
 تکنولوژی‌ها

- Python (Flask)
- SQLite (ذخیره‌سازی محلی تیکت‌ها)
- HTML / CSS / JavaScript (رابط کاربری وب)
- Docker (آماده‌سازی ایمیج برای استقرار روی CaaS)

 قابلیت‌ها

- صدور نوبت جدید برای مشتری
- فراخوانی نوبت بعدی و انتقال به وضعیت **در حال خدمت**
- ثبت پایان خدمت برای نوبت‌های سرویس‌دهی شده
- نگهداری وضعیت‌ها در دیتابیس (Waiting, Serving, Done, …)
- رابط وب مدیریت نوبت‌ها با جدول و نشانگرهای رنگی وضعیت

 ساختار پروژه

  سرویس اصلی Flask (API + UI). این سرویس نقش **Core System** را در معماری نوبت‌دهی دارد.
  شامل قالب‌های HTML (صفحه اصلی و طرح کلی).
  استایل رابط کاربری (پنل مدیریتی نوبت‌ها).
  دیتابیس SQLite برای نگهداری نوبت‌ها (در Git معمولاً نادیده گرفته می‌شود).
  لیست وابستگی‌های پایتون.
  تعریف ایمیج Docker برای اجرای سرویس در محیط کانتینری.


- `POST /api/ticket/new`  
  صدور نوبت جدید و قرار دادن آن در وضعیت Waiting.
- `POST /api/ticket/next` 
  انتخاب اولین نوبت در انتظار و انتقال آن به وضعیت Serving.
- `POST /api/ticket/<number>/done`  
  ثبت پایان خدمت برای نوبت مشخص.
- `GET /api/tickets`  
  دریافت لیست همه نوبت‌ها و وضعیت فعلی آن‌ها (مناسب برای نمایشگر سالن یا مانیتورینگ).

 اجرای محلی
 pip install -r requirements.txt
python server.py
